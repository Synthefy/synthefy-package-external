Authentication

Include your API key in the request headers: 

curl -X GET "https://api.synthefy.com/api/health" \
  -H "X-API-Key: your-api-key-here" \
  -H "Content-Type: application/json"

API Examples

Health Check

curl -X GET "https://api.synthefy.com/api/health" \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json"

Time Series Forecasting

curl -X POST "https://api.synthefy.com/api/forecast" \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "your-user-id",
    "config": {
      "file_path_key": "path/to/your/data.parquet",
      "model_type": "synthefy-forecasting",
      "forecast_length": 30,
      "timeseries_columns": ["value"],
      "timestamp_column": "timestamp",
      "covariate_columns": [],
      "do_llm_explanation": false
    }
  }'

Anomaly Detection

curl -X POST "https://api.synthefy.com/api/zero_shot_anomaly" \

  -H "Content-Type: application/json" \
  -d '{

  }'

Common Endpoints

  • GET /api/health
    Check API health status
  • POST /api/forecast
    Generate time series forecasts
  • POST /api/synthesis
    Generate synthetic time series data
  • POST /api/zero_shot_anomaly
    Detect anomalies in time series data
  • POST /api/train
    Train custom models
  • GET /api/view
    View data and results

Quick Start Guide

  1. Obtain your API key from your account settings
  2. Include the API key in all requests using the X-API-Key header
  3. Test connectivity with the /api/health endpoint
  4. Explore the interactive documentation at /docs
  5. Use the examples above as templates for your requests

Rate Limits

Rate limits vary by plan and endpoint. Check response headers for current limits and remaining quota.
